# YFex.Cqrs.SourceGenerator

A **C# Incremental Source Generator** that automatically generates CQRS static helper methods for DTOs containing nested `Query`, `Command`, and `Events` classes.

---

## What it does

You write this:

```csharp
public partial class UserDto
{
    public static class Query
    {
        public record GetUserByIdQuery(int Id) : IQuery<UserDto>;
    }
    public static class Command
    {
        public record CreateUserCommand(string Name, string Email) : ICommand;
    }
    public static class Events
    {
        public record UserCreatedEvent(int Id, string Name) : IEvent;
    }
}
```

The generator emits `UserDto.g.cs` at compile time:

```csharp
// <auto-generated/>
#nullable enable

public partial class UserDto
{
    public static class Query
    {
        public static System.Threading.Tasks.Task<Result<UserDto>> GetUserById(int id)
        {
            return YFex.Cqrs.Query.Execute(new GetUserByIdQuery(id));
        }
    }

    public static class Command
    {
        public static System.Threading.Tasks.Task<Result> CreateUser(string name, string email)
        {
            return YFex.Cqrs.Command.Execute(new CreateUserCommand(name, email));
        }
    }

    public static class Events
    {
        public static void OnUserCreated(System.Action<UserCreatedEvent> handler)
        {
            // Empty for now - will be implemented later
        }
    }
}
```

## Getting Started

### 1. Reference the generator

```xml
<ItemGroup>
  <ProjectReference Include="..\src\YFex.Cqrs.SourceGenerator.csproj"
                    OutputItemType="Analyzer"
                    ReferenceOutputAssembly="false" />
</ItemGroup>
```

For NuGet (once published):

```xml
<PackageReference Include="YFex.Cqrs.SourceGenerator" Version="1.0.0"
                  PrivateAssets="all" />
```

### 2. See generated files on disk

Add to your consumer `.csproj`:

```xml
<EmitCompilerGeneratedFiles>true</EmitCompilerGeneratedFiles>
<CompilerGeneratedFilesOutputPath>$(BaseIntermediateOutputPath)Generated</CompilerGeneratedFilesOutputPath>
```

Files appear in: `obj/Debug/net8.0/Generated/YFex.Cqrs.SourceGenerator/`

### 3. Profile generator performance

```xml
<ReportAnalyzer>true</ReportAnalyzer>
```

## Detection Rules

The generator scans for:

| Rule | Detail |
|------|--------|
| **Partial class only** | Non-partial classes are ignored |
| **Nested static class named `Query`** | Records implementing `IQuery<T>` inside |
| **Nested static class named `Command`** | Records implementing `ICommand` inside |
| **Nested static class named `Events`** | Records implementing `IEvent` inside |

---

## Generation Rules

### Queries → `Task<Result<T>>`

| Input | Output |
|-------|--------|
| `GetUserByIdQuery(int Id)` | `GetUserById(int id)` |
| `GetUsersByAgeQuery(int MinAge, int MaxAge)` | `GetUsersByAge(int minAge, int maxAge)` |

- Removes `Query` suffix from record name
- Parameters converted to `camelCase`
- Returns `System.Threading.Tasks.Task<Result<T>>`
- Body: `return YFex.Cqrs.Query.Execute(new RecordName(args));`

### Commands → `Task<Result>`

| Input | Output |
|-------|--------|
| `CreateUserCommand(string Name, string Email)` | `CreateUser(string name, string email)` |
| `DeleteUserCommand(int Id)` | `DeleteUser(int id)` |

- Removes `Command` suffix
- Returns `System.Threading.Tasks.Task<Result>`
- Body: `return YFex.Cqrs.Command.Execute(new RecordName(args));`

### Events → `void OnXxx(Action<T> handler)`

| Input | Output |
|-------|--------|
| `UserCreatedEvent(int Id, string Name)` | `OnUserCreated(System.Action<UserCreatedEvent> handler)` |

- Removes `Event` suffix, prepends `On`
- Returns `void`
- Body is intentionally empty (reserved for future pub/sub implementation)

## Architecture

### Incremental Pipeline

```
SyntaxProvider.CreateSyntaxProvider
       │
       ▼ (fast, syntax-only, runs on every keystroke)
IsCandidateClass()
  • Is ClassDeclarationSyntax?
  • Has 'partial' modifier?
  • Has nested class named Query/Command/Events?
       │
       ▼ (slower, semantic, runs only on filtered nodes)
GetClassToGenerate()
  • Resolve INamedTypeSymbol via SemanticModel
  • Find nested static classes
  • Find records implementing IQuery<T>/ICommand/IEvent
  • Extract primary constructor parameters
  • Build ClassToGenerate model
       │
       ▼ (value-equality check — skips if model unchanged)
EquatableArray<T> comparison
       │
       ▼ (only runs when model actually changed)
EmitSource() → {ClassName}.g.cs
```

### Why `CreateSyntaxProvider` instead of `ForAttributeWithMetadataName`?

This generator detects classes **by structure** (nested class names + interface implementation), not by attribute decoration. `ForAttributeWithMetadataName` is 99× faster *but only applies to attribute-based detection*. Since no `[GenerateCQRS]` attribute is required here, `CreateSyntaxProvider` with a tight syntax predicate is the correct choice.

> **Future optimization**: Add an optional `[GenerateCQRS]` marker attribute and switch to `ForAttributeWithMetadataName` for 99× better performance in solutions with 1000+ files.

### Value Equality & Caching

All models (`ClassToGenerate`, `QueryToGenerate`, etc.) implement `IEquatable<T>`. Collections use `EquatableArray<T>` — a structural-equality wrapper — because standard `T[]` and `ImmutableArray<T>` use reference equality, which would invalidate the cache on every pass.
